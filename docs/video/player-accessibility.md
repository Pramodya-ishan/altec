# Player accessibility

The player dialog has an accessible name and modal semantics, video controls expose standard Plyr keyboard behavior, and every custom control has visible text. Captions and transcript data can be displayed when available. Status and error states are text, not color-only signals.

Before enabling production captions, verify WebVTT language labels, keyboard focus containment/return, screen-reader announcements for session errors, contrast in every theme, reduced-motion behavior, and mobile landscape/fullscreen interaction.
